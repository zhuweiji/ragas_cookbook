Question: What is the WSL extension for Visual Studio Code?
Answer: The WSL extension for Visual Studio Code enables running the code editor within the Windows Subsystem for Linux (WSL).

Question: How can you check if WSL is enabled after restarting Windows?
Answer: You can check if WSL is enabled by opening a Command Prompt and typing 'wsl'.

Question: Where can you install Linux distributions for WSL?
Answer: Linux distributions for WSL can be installed from the Microsoft Store.

Question: What command installs Python3 and pip into your Linux installation?
Answer: The command `sudo apt install python3 python3-pip` installs Python3 and pip into your Linux installation.

Question: What does the 'code .' command do in the WSL terminal?
Answer: The 'code .' command launches Visual Studio Code with the current folder.

Question: What notifies the user about connecting to WSL?
Answer: A quick notification telling you that VS Code is connecting to WSL.

Question: What action opens a new terminal instance in WSL?
Answer: Run **Terminal** > **New Terminal** (`kb(workbench.action.terminal.toggleTerminal)`).

Question: What extension recommendation is given when opening a Python file?
Answer: The [Microsoft Python](https://marketplace.visualstudio.com/items?itemName=ms-python.python) extension.

Question: What is the use of setting a breakpoint in the code?
Answer: To inspect variables, create watches, and navigate the call stack during debugging.

Question: How to end a session in WSL and go back to running VS Code locally?
Answer: Using **File** > **Close Remote Connection**.
